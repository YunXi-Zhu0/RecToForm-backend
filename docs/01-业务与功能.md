# 发票信息自助提取填表程序：业务与功能

## 1. 项目概述

### 1.1 项目名称

发票信息自助提取填表程序

### 1.2 项目目标

构建一套基于 FastAPI 的后端服务，对发票文件进行文件类型识别、图像提取与视觉结构化信息抽取，并完成 Excel 自动填表，降低人工录入成本，提高录入效率和一致性。

### 1.3 业务背景

用户需要将发票中的关键信息填写到固定格式的 Excel 表格中。传统方式依赖人工查看票面并逐项录入，效率低且容易出错。本项目通过文件预处理与视觉大模型串联处理，将发票图片直接转换为结构化 JSON，再写入 Excel 模板。

## 2. 用户角色与使用场景

### 2.1 用户角色

- 业务录入人员
- 财务或报销处理人员
- 需要批量整理发票信息的后台操作人员

### 2.2 核心使用场景

- 用户选择默认模板 A 或模板 B。
- 用户查看模板自带的默认字段。
- 当默认字段不足时，用户额外勾选可选字段。
- 用户上传发票文件并发起分析任务。
- 系统输出已填写完成的 Excel 文件。

## 3. 核心业务能力

### 3.1 发票文件上传

系统应支持用户上传发票文件，至少满足以下要求：

- 支持常见图片格式与 PDF。
- 当前至少支持单文件处理。
- 具备后续扩展到多文件批量处理的能力。
- 对文件类型、大小、空文件等异常进行校验。

### 3.2 文件类型判断与预处理

系统应在接收文件后先判断文件类型，并按类型进入不同处理分支。

要求如下：

- 能区分至少 `PDF` 与常见图片文件。
- 图片文件应可直接进入视觉模型输入准备阶段。
- PDF 文件应先进行页面解析或图片抽取，再生成后续分析所需的图片列表。
- 预处理不能破坏票面关键信息、方向、分页关系和清晰度。
- 原始文件与预处理结果应支持留存，便于调试与复核。

### 3.3 PDF 抽图

当上传文件为 PDF 时，系统需将 PDF 转换为可供视觉模型分析的图片输入。

要求如下：

- 保留页码顺序或图片顺序。
- 支持记录每张图片与原始 PDF 页码的对应关系。
- 尽量保留票面原始布局、表格、印章和金额区域。
- 对扫描型 PDF 与内嵌图片型 PDF 都应具备可扩展支持能力。
- 抽图失败时需返回明确错误信息。

### 3.4 模板选择

系统需预置两个默认 Excel 模板供用户选择。

要求如下：

- 每个模板具有唯一标识。
- 每个模板拥有固定默认字段集合。
- 每个模板维护独立的 Excel 单元格映射关系。
- 模板元数据应配置化管理。

### 3.5 可选字段扩展

系统需支持用户在默认模板字段之外，按需勾选可选字段。

要求如下：

- 可选字段可按模板或通用字段池管理。
- 用户勾选结果必须传入后端。
- 后端必须基于所选字段动态调整视觉模型提取目标。
- 仅允许写入存在 Excel 映射位置的可选字段。
- 若字段被勾选但票面不存在，对应结果应返回空值或缺失标记，不得臆造。

### 3.6 qwen3-vl 结构化提取

系统需通过 `qwen3-vl` 模型 API 对发票图片进行字段分析并输出结构化 JSON。

要求如下：

- 输入由系统提示词、用户提示词、模板字段信息和发票图片共同组成。
- 输出必须是 JSON 字符串或可直接解析的 JSON 对象。
- 输出字段必须与模板字段定义一致。
- 不允许以自由文本代替最终结构化结果。
- 对非 JSON、字段缺失、格式异常等情况需要有兜底处理。

### 3.7 JSON 校验与清洗

在写入 Excel 前，系统应对模型输出进行标准化处理。

至少包括：

- JSON 解析校验
- 必填字段校验
- 字段类型校验
- 日期、金额、发票号码等格式校验
- 空值与缺失值标准化
- 冗余字段过滤

### 3.8 Excel 自动填写

系统根据模板字段映射关系，将 JSON 数据写入指定 Excel 模板。

要求如下：

- 支持不同模板对应不同字段布局。
- 写入逻辑必须基于显式映射关系。
- 缺失字段保持空白或按规则填默认值。
- 输出可下载的 Excel 文件。
- 尽量避免破坏模板原有格式。

### 3.9 任务状态与结果返回

系统应向调用方返回明确的任务状态。

建议至少包含：

- 任务创建成功
- 文件预处理中
- PDF 抽图中
- 视觉模型解析中
- Excel 生成中
- 成功
- 失败

失败时应返回尽可能明确的错误信息。

## 4. 输入输出要求

### 4.1 输入参数

建议至少包含：

- 发票文件
- 模板 ID
- 默认字段集合
- 用户勾选的可选字段集合

### 4.2 中间数据

系统内部建议保留：

- 原始上传文件元数据
- 文件类型识别结果
- PDF 抽图结果或图像清单
- LLM 请求提示词
- LLM 原始响应
- 清洗后的 JSON
- Excel 输出文件路径或对象

### 4.3 输出结果

建议至少包含：

- 任务状态
- 结构化提取结果摘要
- Excel 下载信息
- 错误信息

## 5. 风险与约束

### 5.1 文件预处理风险

- 模糊图片、旋转图片、低分辨率 PDF 会影响视觉识别准确率。
- PDF 抽图策略不当可能导致页码顺序混乱或图像质量下降。

### 5.2 LLM 风险

- 视觉模型可能输出非标准 JSON。
- 视觉模型可能猜测不存在的字段，必须通过提示词和校验层限制。

### 5.3 模板风险

- 模板字段定义与 Excel 映射不一致时会导致填表错误。
- 可选字段若无单元格映射，将无法正常落表。

## 6. 验收标准

满足以下条件可视为阶段性验收通过：

1. 用户可选择两个默认模板之一。
2. 用户可在默认模板基础上额外勾选可选字段。
3. 系统可识别上传文件类型，并对 PDF 执行图片抽取。
4. 系统可将发票图片与提示词一并传给 `qwen3-vl` 模型 API。
5. 系统可稳定获得符合约束的 JSON 结果。
6. 系统可根据模板映射将 JSON 写入 Excel。
7. 系统可返回可下载或可获取的结果文件。
8. 异常情况下系统可给出明确错误信息。

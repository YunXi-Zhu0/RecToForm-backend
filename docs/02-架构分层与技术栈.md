# 发票信息自助提取填表程序：架构分层与技术栈

## 1. 技术基线

- Python：`3.9.25`
- 依赖管理：`uv`
- 后端框架：`FastAPI`
- OCR：`PaddleOCR-VL-1.5 API`
- 大模型：`DeepSeek API`
- 项目布局：`src layout`

## 2. 总体分层原则

- API 路由层、业务编排层、外部集成层、配置层必须解耦。
- 外部系统调用必须统一收敛到 `src/integrations/`。
- 业务串联逻辑必须统一收敛到 `src/services/`。
- 路由层不直接拼接外部 API 请求。
- 模板、字段、提示词、Excel 映射尽量配置化。

## 3. 推荐目录结构

### 3.1 API 层

- `src/api/`
  - FastAPI 接口层
- `src/api/routes/`
  - 路由定义
- `src/api/schemas/`
  - 请求模型与响应模型
- `src/api/services/`
  - 异步接口流程与任务调度
- `src/api/app.py`
  - 应用入口、主路由、中间件注册、CORS 配置

### 3.2 核心配置层

- `src/core/`
  - 配置、基础能力、通用工具
- `src/core/config.py`
  - 项目配置主入口
  - 负责统一读取环境变量
  - 负责统一提供服务 `host`、`port`
  - 负责提供外部服务所需配置

### 3.3 集成层

- `src/integrations/`
  - 外部接口封装
- `src/integrations/ocr/`
  - PaddleOCR API client
- `src/integrations/llm/`
  - DeepSeek API client

### 3.4 业务服务层

- `src/services/`
  - 业务流程编排
- `src/services/workflow/`
  - 任务主链路编排
- `src/services/ocr/`
  - OCR 结果标准化与 Markdown 处理
- `src/services/llm/`
  - 提示词构造、响应清洗、JSON 校验
- `src/services/template/`
  - 模板定义、默认字段、可选字段、字段映射
- `src/services/excel/`
  - Excel 模板读取与写入

## 4. API 层架构要求

### 4.1 路由层

- 接收请求参数
- 调用 API service
- 返回标准响应
- 不承担外部 API 细节和复杂业务编排

### 4.2 Schema 层

- 定义请求体
- 定义响应体
- 统一字段命名与序列化结构

### 4.3 API Service 层

- 负责异步请求编排
- 负责接口层任务启动、状态查询、结果返回
- 需要使用 `async` / `await`
- 涉及任务调度时可使用 `asyncio`

### 4.4 FastAPI 应用入口

`src/api/app.py` 内部建议包含：

- 主应用初始化
- 路由注册
- CORS 中间件
- 异常处理中间件或统一异常处理
- 健康检查接口

## 5. 配置管理规范

### 5.1 配置入口

- `src/core/config.py` 是唯一配置主入口。
- 不在不同层重复分散读取环境变量。

### 5.2 敏感信息管理

敏感信息统一放在 `.env` 中，例如：

- API Key
- Token
- 外部服务鉴权配置
- 服务地址

### 5.3 配置加载方式

- 通过 `python-dotenv` 读取 `.env`
- 外部集成 client 统一从 `config.py` 获取配置
- 不在业务代码中散落直接调用 `os.getenv`

## 6. 非功能要求

### 6.1 可维护性

- 各目录职责固定清晰，避免跨层堆叠逻辑。
- 关键流程保留中间结果，便于排查问题。
- 模块化设计优先，避免大而全的单文件实现。

### 6.2 可扩展性

- 支持新增更多 Excel 模板。
- 支持扩展更多可选字段。
- 支持替换 OCR 服务或 LLM 服务。

### 6.3 可观测性

- 关键链路需要日志。
- 应保留请求 ID 或任务 ID。
- 能追踪 OCR Markdown 和 LLM 原始输出。

### 6.4 稳定性

- OCR API 超时、失败、异常响应需有错误处理或重试机制。
- LLM 格式异常需有兜底清洗和校验逻辑。
- Excel 写入失败需返回明确错误。
- 异步任务在高并发场景下应避免阻塞主事件循环。

### 6.5 安全性

- API 密钥通过环境变量管理。
- 日志中不得输出敏感密钥。
- 上传文件需要基础安全校验。
- `.env` 中的敏感数据不得硬编码进入源码仓库。

## 7. Git 与协作规范

### 7.1 Git 提交规范

- Git 提交必须遵循约定式提交规范。
- 所有提交信息必须使用中文。
- 提交说明必须尽量详细，不能只写简短标题。
- 提交说明至少要覆盖：
  - 核心功能
  - 实现思路
  - 关键流程或算法逻辑
  - 影响范围

### 7.2 协作节奏

- 开发过程应保持模块化拆分。
- 每完成一个独立小功能或小阶段，都应主动确认是否需要提交。
- 未经用户明确要求时，不擅自创建提交。

## 8. 配置项建议

建议配置化以下内容：

- OCR API 地址与鉴权参数
- DeepSeek API 地址与模型参数
- FastAPI 服务 `host`
- FastAPI 服务 `port`
- 模板定义
- 默认字段定义
- 可选字段定义
- 字段到 Excel 单元格的映射关系
- 提示词模板
